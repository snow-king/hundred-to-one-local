# 100 к 1 — локальная веб-версия

Локальная браузерная версия игры "100 к 1" для 4 отдельных игроков.

Игра работает полностью на клиенте: без сервера, без БД, с сохранением состояния в `localStorage`.

## Стек

- Vue 3 + TypeScript
- PrimeVue + PrimeIcons
- Tailwind CSS v4 (через `@tailwindcss/vite`, без PostCSS)
- Vite

## Что реализовано

- 4 индивидуальных игрока (не команды)
- Роль модератора/ведущего
  - выбирает, кто отвечает
  - вводит ответ
  - система проверяет совпадение
- Начисление очков только при точном совпадении ответа
- Сообщение "Нет такого слова" при промахе
- Раунд завершается, когда открыты все ответы
- Итоговый экран с победителем и таблицей
- Автосохранение игры в браузере
- Восстановление сохраненной игры
- Лента последних попыток
- Несколько пакетов вопросов из коробки
- Выбор пакета перед стартом
- Архитектура без ограничений по числу пакетов (добавляете новые в JSON)
- Анимация открытия ответа (flip)
- Режимы экрана для ТВ/второго дисплея
  - общий вид
  - только табло
  - только пульт
- Синхронизация между вкладками по `storage`-событию

## Режимы отображения (URL)

Параметр `view` управляет отображением.

- `?view=split` — общий вид (по умолчанию)
- `?view=board` — только табло (удобно вывести на ТВ)
- `?view=control` — только пульт ведущего

Пример:

- `http://localhost:5173/?view=board`
- `http://localhost:5173/?view=control`

Режим можно переключать кнопками в интерфейсе, URL обновляется автоматически.

## Сценарий "пульт + табло"

1. Откройте вкладку/окно A: `?view=control`
2. Откройте вкладку/окно B: `?view=board`
3. Вводите ответы в A — изменения автоматически появятся в B

Важно: вкладки должны быть в одном браузере и на одном origin (протокол + хост + порт).

## Локальное хранение

- Ключ: `hundred-to-one:v1`
- Сохраняется:
  - статус игры
  - текущий вопрос
  - открытые ответы
  - очки игроков
  - выбранный игрок
  - история попыток
- При сбросе игра удаляется из `localStorage`

## Пакеты вопросов

Данные находятся в `src/data/packs.json`.

Формат:

```json
[
  {
    "id": "classic",
    "title": "Классика",
    "description": "Описание пакета",
    "questions": [
      {
        "id": 1,
        "prompt": "Текст вопроса",
        "answers": [
          { "text": "Ответ 1", "points": 32 },
          { "text": "Ответ 2", "points": 24 }
        ]
      }
    ]
  }
]
```

Количество пакетов не ограничено: добавляйте новые элементы в массив.

Проверка ответа выполняется с нормализацией:

- без учета регистра
- лишние пробелы схлопываются
- `ё` приравнивается к `е`

## Ограничения текущей версии

- Нет серверной части и аккаунтов
- Нет редактирования пакета вопросов через UI (только JSON в проекте)
- Синонимы не поддерживаются автоматически (только точные совпадения после нормализации)
