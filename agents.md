# План реализации локальной веб-версии "100 к 1"

## Цель
Сделать браузерную офлайн-игру "100 к 1" для 4 отдельных игроков на стеке Vue + TypeScript + PrimeVue + Tailwind. Состояние игры хранится в браузере.

## Принятые правила
- Никаких команд: только 4 индивидуальных игрока.
- Модератор управляет раундом вручную.
- Модератор выбирает активного игрока и вводит его ответ.
- Если ответ найден в списке вопроса, система открывает его и начисляет очки выбранному игроку.
- Если ответа нет, показывается сообщение: "нет такого слова".
- Раунд завершается, когда открыты все ответы к вопросу.
- После последнего вопроса показывается итоговая таблица и победитель.

## Данные
- Вопросы/ответы хранятся в статическом JSON внутри проекта.
- Модель игры включает:
  - игроков (id, имя, счет),
  - список вопросов,
  - индекс текущего вопроса,
  - выбранного игрока,
  - историю попыток,
  - статус игры (setup/playing/finished).

## Экраны
1. Старт:
   - ввод имен 4 игроков,
   - запуск новой игры,
   - продолжение сохраненной игры.
2. Игра:
   - текущий вопрос,
   - список ответов (скрытые/открытые),
   - панель модератора (выбор игрока + ввод ответа + проверка),
   - таблица очков,
   - последние попытки.
3. Итоги:
   - победитель,
   - финальная таблица,
   - новая игра / быстрый рестарт тем же составом.

## Локальное хранение
- Автосохранение состояния после каждого значимого действия.
- Хранение в localStorage под ключом с версией (`hundred-to-one:v1`).
- Восстановление игры при повторном открытии страницы.

## UX требования
- Ясный интерфейс модератора.
- Проверка ответов с нормализацией (регистр, лишние пробелы, е/ё).
- Уведомления об успехе/ошибке.
- Адаптивная верстка для desktop и mobile.
